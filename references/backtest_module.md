# 因子回测模块设计

## 使用时机

当用户要求“回测方案”“IC/RankIC 检验”“分层测试”“多空组合测试”“下一阶段怎么验证因子”时读取本文件。

当前版本只设计回测方案，不提供自动化脚本，不生成代码，不运行回测。

## 产物

```text
factor_backtest_plan.md
```

## 通过标准建议

| 结果 | 判断 |
|---|---|
| RankIC 均值稳定为正或负，且方向符合假设 | 可进入下一阶段 |
| 分层收益单调，多空组合成本后为正 | 可进入策略候选 |
| 只在少数年份有效 | 降级为状态因子 |
| 控制行业/市值后失效 | 降级或放弃 |
| 使用真实可得时间后失效 | 判定未来函数风险，停止 |

## 关键约束

- 先验证时点正确，再看收益。
- 先做简单可解释版本，再做复杂模型版本。
- 不因单次好结果直接进入实盘或策略生成。
- 所有结论写成“测试方案”或“待验证标准”，不要写成已验证事实。
